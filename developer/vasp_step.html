The control parameters for Energy.

You need to replace the ‚Äútime‚Äù entry in dictionary below these comments with the definitions of parameters to control this step. The keys are parameters for the current plugin,the values are dictionaries as outlined below.

Examples

parameters = {
    "time": {
        "default": 100.0,
        "kind": "float",
        "default_units": "ps",
        "enumeration": tuple(),
        "format_string": ".1f",
        "description": "Simulation time:",
        "help_text": ("The time to simulate in the dynamics run.")
    },
}
parameters{str: {str: str}}

A dictionary containing the parameters for the current step. Each key of the dictionary is a dictionary that contains the the following keys:

parameters[‚Äúdefault‚Äù] :

The default value of the parameter, used to reset it.

parameters[‚Äúkind‚Äù]enum()

Specifies the kind of a variable. One of ‚Äúinteger‚Äù, ‚Äúfloat‚Äù, ‚Äústring‚Äù, ‚Äúboolean‚Äù, or ‚Äúenum‚Äù

While the ‚Äúkind‚Äù of a variable might be a numeric value, it may still have enumerated custom values meaningful to the user. For instance, if the parameter is a convergence criterion for an optimizer, custom values like ‚Äúnormal‚Äù, ‚Äúprecise‚Äù, etc, might be adequate. In addition, any parameter can be set to a variable of expression, indicated by having ‚Äú$‚Äù as the first character in the field. For example, $OPTIMIZER_CONV.

parameters[‚Äúdefault_units‚Äù]str

The default units, used for resetting the value.

parameters[‚Äúenumeration‚Äù]: tuple

A tuple of enumerated values.

parameters[‚Äúformat_string‚Äù]: str

A format string for ‚Äúpretty‚Äù output.

parameters[‚Äúdescription‚Äù]: str

A short string used as a prompt in the GUI.

parameters[‚Äúhelp_text‚Äù]: str

A longer string to display as help for the user.

See also

Energy, TkEnergy, Energy, EnergyStep

class vasp_step.energy_step.EnergyStep(flowchart=None, gui=None)[source]#

Bases: object

Helper class needed for the stevedore integration.

This must provide a description() method that returns a dict containing a description of this node, and create_node() and create_tk_node() methods for creating the graphical and non-graphical nodes.

The dictionary for the description is the class variable just below these comments. The felds are as follows:

my_description{str, str}

A human-readable description of this step. It can be several lines long, and needs to be clear to non-expert users. It contains the following keys: description, group, name.

my_description[‚Äúdescription‚Äù]tuple

A description of the Energy step. It must be clear to non-experts.

my_description[‚Äúgroup‚Äù]str

Which group in the menus to put this step. If the group does not exist it will be created. Common groups are ‚ÄúBuilding‚Äù, ‚ÄúControl‚Äù, ‚ÄúCustom‚Äù, ‚ÄúData‚Äù, and ‚ÄúSimulations‚Äù.

my_description[‚Äúname‚Äù]str

The name of this step, to be displayed in the menus.
